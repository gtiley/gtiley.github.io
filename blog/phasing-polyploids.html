Phased Haplotype Sequences from Polyploids

12 June 2021

Why should data be phased?

I originally became interested in phasing sequence data for polyploids when I was learning about genotyping for accurate estimation of site frequency spectra for various grass species in Madagascar. At the same time, I was applying the MSci model 1 for divergence time estimation with introgression for the mouse lemur work, and it seemed like phasing data would be a good strategy for enabling easier analyses of polyploids with network models. I was recently involved in some generation and analysis of target enrichment data 2, so that seemed like a good place to start seeing as these data are arguably more accessible than other types of genomic data.
Honestly, I needed to convince myself that phasing actually mattered for improving analyses of polyploids, specifically allopolyploids. There were a few empirical examples that seemed important 3 4, but the methods were not easily scalable for many individuals. So I first started with some simulations under a simple scenario with a single allopolyploid (Fig. 1). Simulations were done under the MSC with very simple assumptions about rate variation, pop sizes, and the substitution model.

Fig. 1 — Species network used for simulation. The divergence times in expected substitutions per site are given for each node, and h is the hybrid node where two alleles enter from both t and v. E is an allotetraploid while other species are diploid. Nucleotide divergence was reduced by dividing all τ by 10 or increased by multiplying all τ by 10. ILS was increased by halving the distance between τ_h and τ_u and between τ_u and τ_s either once (for the low ILS condition) or twice (for the moderate ILS condition).

Networks were then estimated using a topology-based pseudolikelihood approach with PhyloNetworks 5. The results were motivating enough that phasing should be useful for investigating at least allopolyploids. Five different input data sets were prepared for the analysis:

  1. True - the known simulated gene trees
  2. Phased - gene trees estimated from the true haplotype data
  3. Genotype - gene trees estimated from sequences where variant sites (within an individual) are collapsed into their IUPAC codes. This would be the result of using some genoytping software without phasing.
  4. Consensus - gene trees based on sequences where only one base per variant site is used.
  5. Pick One - gene trees estimated with the tye haplotype data, but only 1 allele is randomly selected per individual. The phased data greatly outperformed haploype consensus data and genotype data for accurately estimating a network (Fig. 2). It is also important to note that a decent number of loci are needed for network estimation. I remember starting out in molecular evolution by cloning Sanger loci in polyploids. Cloning is a lot of work and I have seen other approached for obtaining lower numbers of long phased loci advocated for, but I am not convinced this is the best way forward given sampling error - especially for high ILS scenarios.

Fig. 2 — Proportion of simulations that correctly identify the allopolyploid lineage. The x-axes are the number of loci sampled for each simulation. Th y-axes are the proportions of correct networks. Results are based on networks estimated with a single reticulation, even if that network was considered less optimal than networks with zero or two reticulations. We saved the true gene trees from the simulations, while we estimated gene trees with the phased and unphased (Genotype, Consensus, and Pick One) data.

A fast and scalable phasing strategy

Along with my coauthor Andy Crowl, we developed a pipeline (Fig. 3) called PATÉ (Phased Alleles from Target Enrichment data). The core of PATÉ is H-PoPG 6. Currently, de novo haplotype consensus assemblies are used at a reference for each individual. Reads are then realigned to loci for genotyping and those genotypes go through some filtering. H-PoPG is then used to phase those variants passing filters, and phased fasta files are output for analysis. I think this is a good strategy for analyses of multiple species where there is no single reference genome, but some power is lost compared to joint genotyping if you have a reference genome and are more interested in population genomics.

Fig. 3 — PATÉ Phasing Pipeline. Overview of data input, output, and steps taken to phase alleles. Input data are assumed to be paired-end Illumina reads and reference sequences for each individual are required (consensus loci from HybPiper can be used). The ploidy of each individual must be specified. Only biallelic sites are used for phasing.

What I like most about PATÉ is that were are able to be rigorous about genotype quality while also being fast. And because we operate on individuals, the genotyping and phasing can be distributed easily on a cluster to scale to many individuals as is typical of contemporary phylogenomics projects.

We have analyzed several empirical data sets to-date using PATÉ, but the one from the preprint are Dryopteris ferns 7. We chose this group because the parents for the allopolyploids are assumed known and because the system is actually quite complex. We do a number of analyses covered in the preprint that suggest the phased data is important for network estimation, especially when there are multiple introgression events involved. Some good news is that when scenarios are relatively simple (i.e. two parents and their allopolyploid), both phased and the haplotype consensus data can do well (Fig. 4). Perhaps alarmingly though, the haplotype consensus data can get the direction of introgression wrong at an alarming rate when there are too few loci.

Fig. 4 — Network search results for three-taxon Dryopteris example. Both phased and unphased haplotype consensus data recovered the same network topology with introgression occurring in the expected direction. The major topology is indicated by the blue edge and the minor edge (direction of introgression) is shown in tan. The inheritance probability γ was slightly higher in the haplotype consensus data (γunphased = 0.455). Bar plots show the proportion of 100 replicates when sampling four or 40 loci that correctly detect one reticulation based on the pseudolikelihood scores (top), correctly estimate the network with introgression going from one of the diploids into D. campyloptera (middle), and estimate a network where the direction of introgression is from D. campyloptera into one of the diploids (bottom).

There might be scenarios where phasing is a good option. I think the haplotype consensus data should be fine for analyzing really ancient events. I advocate that the phasing might be beneficial for tackling more recent species complexes though.
